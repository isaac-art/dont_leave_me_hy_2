# HysteriaManager — iOS

A native iOS app that manages an **on-device VPN** for your Hysteria 2 servers:
import multiple configs (hysteria2 links or **QR scan**), auto-pick the **fastest**,
and route **China direct / everything else through hysteria** — Clash-style — using
**sing-box** inside a Network Extension.

> Sibling of the macOS app in `../HysteriaManager`. Shared, platform-agnostic code
> lives in `../Shared` (profile model + URI parser).

---

## How it works

```
 all device traffic ─► TUN (Network Extension) ─► sing-box engine
                                                      │
                                       ┌──────────────┴───────────────┐
                              geoip:cn / geosite:cn / private     everything else
                                       │                               │
                                    direct                     hysteria2  (urltest = fastest)
```

- iOS can't run a CLI or set a system proxy, so routing happens inside a
  **`NEPacketTunnelProvider`** extension running **sing-box** (`Libbox.xcframework`).
- The app turns your hysteria2 profiles into a sing-box config
  (`Core/SingboxConfigBuilder.swift`): one `hysteria2` outbound per profile, a
  `urltest` outbound that auto-selects the **fastest**, and `route` rules that send
  `geoip:cn` + `geosite:cn` + private IPs to **direct** and the rest to the proxy.
- The app writes that config to the **App Group** container; the extension loads it.

## Features

- Import via **hysteria2:// / hy2:// links** (paste, bulk, or open-in), or **QR scan**.
- **Auto (fastest)** across all enabled profiles, or **Manual** pick.
- Global routing mode: **Rule-based (CN direct)** / **Global** / **Direct**.
- One-tap **Connect/Disconnect**, live status, **connect-on-launch**, and
  **on-demand** auto-reconnect on network changes.
- Settings for the URL-test target/interval.
- **Tailscale (optional, toggleable)**: join your tailnet *inside the same tunnel* —
  reach your other devices by their `100.x` address while the internet still routes
  through hysteria2 (one iOS tunnel, so no "two VPN apps can't both run" problem).
  Settings → Tailscale → on + an auth key (Tailscale admin → Settings → Keys). Tailnet
  range `100.64.0.0/10` routes to the Tailscale endpoint; everything else is unchanged.
  No Libbox rebuild needed (`with_tailscale` is in the Apple build). Notes: `tsnet` adds
  memory pressure to the extension (watch the Tunnel Log if flaky); MagicDNS name
  resolution (`*.ts.net`) is **not** wired up yet — reach peers by IP for now (the
  modern DNS format it needs caused connect failures, so it's parked).

## Requirements

- **Xcode 16+**, iOS **17+** device.
- A **paid Apple Developer account** (Network Extensions need it — no way around it).
- **`Libbox.xcframework`** (sing-box for iOS) in `Frameworks/` — see
  `Frameworks/PUT_LIBBOX_XCFRAMEWORK_HERE.md` for the gomobile build command.

## Build & run

A ready-to-open **`HysteriaManagerMobile.xcodeproj`** is included — no XcodeGen needed.

```sh
cd HysteriaManagerMobile
# 1. build Libbox.xcframework → Frameworks/  (see Frameworks/PUT_LIBBOX_XCFRAMEWORK_HERE.md)
brew install go
# 2. open and build
open HysteriaManagerMobile.xcodeproj
```

> `project.yml` is also kept as the source of truth. If you change the file layout,
> you can regenerate the project with `brew install xcodegen && xcodegen generate`.

In Xcode:

1. Select both targets → **Signing & Capabilities** → set your **Team**
   (or set `DEVELOPMENT_TEAM` in `project.yml` and re-run `xcodegen`).
2. Confirm both targets have:
   - **App Groups** → `group.com.isaacclarke.HysteriaManager`
   - **Network Extensions** → *Packet Tunnel* (already in the generated `.entitlements`)
   - Bundle IDs: app `com.isaacclarke.HysteriaManagerMobile`,
     extension `…​.tunnel` (must match `VPNManager.tunnelBundleID`).
3. Run on a **real device** (the Simulator can't route real traffic).
4. First connect installs the VPN profile (iOS prompts) and asks for the config
   permission. Then: Import a link / scan a QR → **Connect**.

## Version note (important)

The `Libbox…Protocol` method set is generated by gomobile and **changes between
sing-box versions**. The Swift glue in `Tunnel/PlatformInterface.swift` +
`PacketTunnelProvider.swift` matches **sing-box v1.13.13**, which is what
`build-libbox.sh` pins by default (`SINGBOX_REF=v1.13.13`).

⚠️ Do **not** build Libbox from sing-box's default `testing` branch — its API differs
(renamed methods, `getDNSServerAddress` returns an iterator, extra shell/tailscale
methods) and the app won't compile. If you bump to a newer sing-box, reconcile the
protocol method names with that version's `experimental/libbox/platform.go`.
